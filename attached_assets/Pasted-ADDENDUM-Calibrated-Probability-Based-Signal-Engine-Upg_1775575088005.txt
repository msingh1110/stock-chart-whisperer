ADDENDUM: Calibrated Probability-Based Signal Engine

Upgrade the app’s signal logic from simple binary scoring to a more robust probability-based directional model that outputs:

- Up probability
- Down probability
- Final signal (BUY / SELL / HOLD)

This should feel stronger, more realistic, and less arbitrary than equal-weight rule counting.

Requirements:

1. Core scoring philosophy
- Start from a neutral baseline
- Use a continuous weighted score instead of simple +1 / -1 counting
- Let stronger signals contribute more than weaker ones
- Convert the final score into probabilities using a logistic / sigmoid function

2. Indicators to use
For each ticker, compute:
- current price
- MA20
- MA50
- RSI (14)
- short-term momentum
- current volume
- average volume
- previous close

3. Weighted scoring logic

Create a continuous `score` variable initialized to 0.

A. Trend: price vs MA20
- Compute:
  - trend_short = (price - MA20) / MA20
- Add weighted contribution:
  - score += 2 * trend_short

B. Trend: MA20 vs MA50
- Compute:
  - trend_long = (MA20 - MA50) / MA50
- Add weighted contribution:
  - score += 2 * trend_long

C. RSI contribution
- If RSI is between 50 and 65:
  - add +1
- If RSI > 70:
  - add -1 because the stock may be overbought
- If RSI < 45:
  - add -1 because the trend is weak / bearish
- Otherwise:
  - add 0

D. Momentum contribution
- Compute short-term momentum as:
  - momentum = (price - price_5d_ago) / price_5d_ago
- Add weighted contribution:
  - score += 1.5 * momentum

E. Volume confirmation
- Compute:
  - volume_ratio = current_volume / average_volume
- If price > previous_close:
  - score += 1 * (volume_ratio - 1)
- Else if price < previous_close:
  - score -= 1 * (volume_ratio - 1)
- Else:
  - add 0

4. Probability conversion
After computing the final score, convert it into probabilities using a sigmoid / logistic function:

- up_probability = 1 / (1 + exp(-score))
- down_probability = 1 - up_probability

This should ensure:
- probabilities are smooth
- probabilities sum to 1
- stronger scores create stronger confidence

5. Final signal mapping
Map probabilities into signals:
- BUY if up_probability >= 0.70
- SELL if down_probability >= 0.70
- HOLD otherwise
